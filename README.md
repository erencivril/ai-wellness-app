# AI Wellness Coach

A health and wellness mobile application where users interact with AI coaches (Dietitian, Fitness Coach, Pilates Instructor, Yoga Teacher). Built for the Connectinno Flutter Academy case study.

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Flutter |
| State Management | Cubit (`flutter_bloc`) |
| AI Backend | Firebase AI — Gemini 2.5 Flash (`firebase_ai`) |
| Dynamic Config | Firebase Remote Config |
| Local Storage | SQLite (`sqflite`) |
| Routing | `go_router` |
| DI | `get_it` |
| Typography | Fraunces + Plus Jakarta Sans (`google_fonts`) |

---

## Architecture

```
lib/
  app/                    # MaterialApp, theme
  core/
    constants/
    database/             # AppDatabase (sqflite singleton)
    di/                   # get_it service locator
    router/               # GoRouter with StatefulShellRoute
    services/             # FirebaseAIService, RemoteConfigService
  features/
    coaches/              # Coach list, cards
    chat/                 # Chat session, streaming AI responses
    history/              # Past conversation list
```

**Key decisions:**

- **Coach list is hardcoded; only system instructions are dynamic.** The 4 coaches are stable product decisions — only their AI personas need to change at runtime. This means the coaches screen loads instantly without a network dependency.
- **`last_message` and `coach_name` are denormalized on the session row.** The history screen needs these fields for every row. Storing them directly avoids a JOIN or N+1 query.
- **`get_it` over provider-based DI.** Services and repositories must be accessible outside the widget tree (e.g. in Cubits). `get_it` handles this cleanly.
- **`StatefulShellRoute.indexedStack`** preserves scroll position and navigation state independently per tab.
- **Resuming conversations:** Persisted messages are loaded from SQLite, converted to `List<Content>`, and passed as `history` to `model.startChat()`. The AI receives full prior context.

## Running the App

### Prerequisites

- Flutter SDK ≥ 3.32
- A Firebase project with **Firebase AI** and **Remote Config** enabled

### 1. Clone the repo

```bash
git clone https://github.com/erencivril/ai-wellness-app.git
cd ai-wellness-app
```

### 2. Create a Firebase project

1. Go to [Firebase Console](https://console.firebase.google.com) and create a new project.
2. Enable **Firebase AI** (Gemini Developer API) in your project.
3. Enable **Remote Config**.
4. Add an **Android** app (package name: `com.connectinno.ai_wellness_app`) and/or an **iOS** app (bundle ID: `com.connectinno.aiWellnessApp`).
5. Download `google-services.json` (Android) and/or `GoogleService-Info.plist` (iOS).

### 3. Install FlutterFire CLI and configure Firebase

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This generates `lib/firebase_options.dart` (gitignored). Place it in `lib/`.

### 4. Add Firebase config files

```
android/app/google-services.json        ← download from Firebase Console
ios/Runner/GoogleService-Info.plist     ← download from Firebase Console
lib/firebase_options.dart               ← generated by flutterfire configure
```

### 5. Set up Remote Config

In Firebase Console → Remote Config, add the following string parameters:

| Key | Default value (example) |
|---|---|
| `dietitian_system_instruction` | `You are a registered dietitian. Provide evidence-based nutritional advice. Be empathetic and practical.` |
| `fitness_coach_system_instruction` | `You are a certified personal trainer. Help users with workout plans and motivation. Prioritize safety.` |
| `pilates_instructor_system_instruction` | `You are a certified Pilates instructor. Guide users through Pilates principles and mindful movement.` |
| `yoga_teacher_system_instruction` | `You are an experienced yoga teacher. Share guidance on yoga poses, breathing, and meditation.` |

Publish the changes after adding the keys.

> **Note:** The app ships with fallback default values for all keys, so it works even before Remote Config is fetched.

### 6. Install dependencies and run

```bash
flutter pub get
flutter run
```

## Features

- **Coaches screen** — Grid of 4 AI coaches, each with a unique identity color and persona loaded from Remote Config.
- **Chat screen** — Real-time streaming responses from Gemini 2.5 Flash. Typing animation with a blinking cursor while the AI is generating.
- **Chat history** — All sessions persisted locally via SQLite. Tap any session to resume it — the AI receives the full prior conversation as context.
